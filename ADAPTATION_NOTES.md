# 数学教材适配笔记

> 本文是 `math-textbook-to-skill` 的原始设计文档。
> 基于《线性代数应该这样学》(Sheldon Axler) 的实战适配经验。

原始适配笔记保存在：[ADAPTATION_NOTES.md](./ADAPTATION_NOTES.md)

## 为什么数学教材需要单独适配

原始 book-to-skill 是为「方法论/决策框架/清单/原则/概念体系」类书籍设计的。而数学教材的特征完全不同：

- 知识结构是 **「定义 → 定理 → 证明 → 例题 → 习题」** 的线性链
- 概念之间存在严格的 **前置依赖**
- 单个定理不能独立使用，必须嵌在依赖图中
- "行动步骤"变成了"证明模式"和"解题方法"
- 没有"作者旧应用（A1）"——有的是**典型例题**
- 没有"未来触发（A2）"——有的是**定理适用条件**

## 六个阶段的主要改动

### 阶段 0：新增 3 项产出
- 十大核心结果（全书最重要的 10 条定理）
- 预备知识清单
- 证明路径图

### 阶段 1：7 个提取器
- 定理提取器（改编自原则提取器）
- 例题提取器（改编自案例提取器）
- 反例/边界提取器（保留核心）
- 术语提取器（保留并增强）
- 证明模式提取器（改编自框架提取器）
- 依赖图提取器（新增，数学教材特有）
- 习题分类器（新增，数学学习核心）

### 阶段 1.5：验证标准重定义
- 定理验证：至少被另一章的定理引用过
- 通过率从 25-50% 提升到 85-100%
- 验证主要价值从「淘汰」变为「去重 + 标注依赖重要性」

### 阶段 2：RIA++ → 数学版
- 概念速查（替代 R）→ 表格形式速查卡片
- 核心定理（替代 I）→ 编号 + 一句话概括
- 典型例题（替代 A1）→ 解题步骤 1-2-3
- 常见错误（替代 B）→ ❌ 为什么错 + 正确做法
- 相关技能（保留 Zettelkasten 链接）
- 去掉 A2（未来触发）和 E（可执行步骤）

### 阶段 3：4 种关系
- A 依赖 B / A 使用 B / A 对比 B / A 推广 B

### 阶段 4：4 类测试
- 直接提问 (50%) / 边界模糊 (30%) / 诱饵 (10%) / 应用场景 (10%)
- 特别关注 F=R 和 F=C 域区分

完整适配说明见 `methodology/` 目录下各阶段文档。
