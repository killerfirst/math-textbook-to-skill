# 阶段 2 — 数学 RIA++ 构造 skill

## 目标

把阶段 1.5 通过的每个语义单元，构造成符合 Claude Code skill 规范的 SKILL.md。

使用模板：`templates/SKILL.md.template`（数学版）

## 数学版五段结构

### 段 1 — 概念速查（Concept Quick Reference）

替代通用版的 R（原文引用）。

数学教材不需要"原文引用"——需要的是**快速查概念**。

```markdown
## 概念速查

| 概念 | 定义 | 关键点 |
|------|------|--------|
| 向量空间 | 满足 8 条公理的集合 V | 加法封闭、数乘封闭 |
| 子空间 | 向量空间的非空子集，自身是向量空间 | 必须包含 0 |
| 线性相关 | 存在一组不全为零的标量使组合为零 | 至少一个向量可被其余表示 |
```

每个核心概念 ≤ 3 行。目的是让用户一眼扫到。

### 段 2 — 核心定理（Core Theorems）

替代通用版的 I（Interpretation）。

```markdown
## 核心定理

| 编号 | 定理名 | 一句话 |
|------|--------|--------|
| 2.14 | 维数公式 | dim V = dim null T + dim range T |
| 3.20 | 线性映射矩阵表示 | 每个线性映射对应一个矩阵 |
```

定理 ID + 一句话，方便扫描定位。

### 段 3 — 典型例题（Typical Examples）

替代通用版的 A1（Past Application）。

```markdown
## 典型例题

### 例 1：判断集合是否为子空间
**题目**：判断 W = {(x₁,x₂,x₃)∈F³ : x₁+2x₂=0} 是否 F³ 的子空间

**解**：
1. 验证 0 ∈ W：0+2·0=0 ✓
2. 验证加法封闭：设 u,v∈W，则 (u₁+2u₂)=0, (v₁+2v₂)=0，相加得 (u₁+v₁)+2(u₂+v₂)=0 ✓
3. 验证数乘封闭：设 a∈F，则 a(u₁+2u₂)=a·0=0 ✓

**教学意图**：展示判断子空间的"三步法"
```

### 段 4 — 常见错误（Common Mistakes）

替代通用版的 B（Boundary）。

```markdown
## 常见错误

- ❌ "零向量就是 0" —— 零向量是**加法的单位元**，不是数字 0
  → 正确：每个向量空间有唯一的加法单位元
- ❌ "子空间必须包含 0 就够了" —— 还需要封闭性
  → 正确：子空间 = 非空子集 + 加法封闭 + 数乘封闭
```

## 为什么这样改动

| 通用版段 | 去掉原因 | 替换 | 替换原因 |
|---------|---------|------|---------|
| R（原文引用） | 数学定理必须精确陈述，不能压缩到 ≤150 字 | 概念速查表格 | 用户最常用场景是查定义 |
| I（interpretation） | 数学定理不能"用自己的话重写" | 核心定理表 | 精确性优先 |
| A1（书中案例） | 数学的"案例"是例题 | 典型例题 | 解题模式比案例更有用 |
| A2（未来触发） | 数学通过概念名触发，不是"场景触发" | （集成到 frontmatter） | 触发词 = 定理名/概念名 |
| E（可执行步骤） | 数学没有"执行"概念 | （集成到例题步骤） | 解题步骤本身就是 E |
| B（边界条件） | 数学的边界是反例 | 常见错误 | 学习者的真正痛点 |

## Frontmatter 设计

```yaml
---
name: subspaces
description: | 
  用户提到向量空间、子空间、线性组合、张成空间等概念时激活。
  特别适用于：判断一个集合是否是子空间、计算子空间的和与交。
  不适用于：线性映射、特征值等其它章节内容。
tags: [linear-algebra, vector-space, subspace, spanning-set]
related_skills: []
---
```

## 常见失败模式

1. **概念速查写成了教科书复述**——不是抄定义，而是提炼"考试前扫一眼"的速查表
2. **核心定理忘了标依赖**——不标依赖关系，skill 就只是孤立的定理卡片
3. **例题步骤跳步**——"容易看出"是数学书误导；要真的把每一步写出来
4. **常见错误太泛**——"不要粗心"不算常见错误；"忘记验证零向量是否在子空间中"才是
