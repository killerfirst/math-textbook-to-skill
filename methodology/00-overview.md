# RIA-TV++（数学版）方法论总览

本文是 math-textbook-to-skill 所用 SOP 的设计说明。执行时的具体步骤请读 `SKILL.md` 和各阶段文档。

## 与通用版的核心差异

math-textbook-to-skill 不是对 cangjie-skill 的简单 prompt 调整，而是**整套方法论的重构**——因为知识单元的性质完全不同。

| 维度 | cangjie-skill（通用版） | math-textbook-to-skill（数学版） |
|------|------------------------|--------------------------------|
| 知识单元 | 原则 → 行动步骤 | 定义 → 定理 → 证明 → 例题 |
| 单元关系 | 松散耦合 | 严格前置依赖 |
| 提取器 | 5 个 | 7 个 |
| RIA++ 结构 | R-I-A1-A2-E-B | 概念速查-核心定理-典型例题-常见错误-相关技能 |
| 验证通过率 | 25-50% | 85-100%（去重为主） |
| 核心产出 | 可执行原则 | 可调用的推理工具 |

## 思想来源

| 来源 | 借鉴内容 |
|------|---------|
| Mortimer Adler《如何阅读一本书》 | 阶段 0：分析阅读三阶段 |
| cangjie-skill | 整体流水线框架 + 并行提取 + 三重验证 + darwin 兼容 |
| 数学教育研究（Polya / Halmos） | 证明模式的概念、例题-习题的教学结构 |
| Niklas Luhmann Zettelkasten | 原子化 + 链接 + 4 种关系类型 |
| darwin-skill | test-prompts.json 格式 + 可进化性 |

## 不变量

1. **定理原子性**：一个 skill 包含一个语义单元（通常为 1-2 章内容），不能是截断的半个定理
2. **可追溯**：每个定理必须标注编号 + 出处章节
3. **可验证**：每个 skill 必须通过三重验证 + 压力测试
4. **可进化**：每个 skill 必须附带 darwin 兼容的 test-prompts.json
5. **依赖显式**：所有前置依赖必须在 INDEX.md 中标注
